# views

Здесь живут страницы приложения — по одной директории на маршрут.

Каждый view самодостаточен: внутри может быть своя `ui/`, `model/`, `api/`, `utils/`. Доменная логика по умолчанию не выносится наружу — она остаётся в том view, где нужна. Если логика становится нужна нескольким страницам, она переезжает в `/entities`.

**Типичная структура view:**

```
room-view/
  ui/       — компоненты страницы
  model/    — локальное состояние, хранилища
  api/      — запросы и подписки, специфичные для этой страницы
  utils/    — вспомогательные функции
```
